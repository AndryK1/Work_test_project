**Тестовый проект** — это учебное Android-приложение, разработанное в соответствии с техническим заданием. Проект демонстрирует современный подход к разработке: многомодульность, чистая архитектура, внедрение зависимостей через Koin, работа с локальными данными (Room) и настройка сетевого слоя (Retrofit + интерсептор). На данный момент проект находится в активной разработке, но уже показывает ключевые навыки, важные для коммерческой разработки.

---

## ✨ Что уже реализовано (и что показывает мой код)

- 🧱 **Многомодульная архитектура** — код разделён на независимые модули (data, di, domain, ui, utils), что обеспечивает масштабируемость и переиспользуемость.
- 🏛 **Clean Architecture + MVVM** — чёткое разделение ответственности между слоями: data (работа с БД и сетью), domain (бизнес-логика),di(зависимости) UI, (ViewModel и фрагменты), utils(утилитарные классы). Код легко тестировать и поддерживать.
- 💉 **DI через Koin** — все зависимости управляются с помощью Koin (модули определены для каждого слоя), что делает код слабосвязанным.
- 🗄 **Локальное хранилище Room** — реализована запись и чтение данных из локальной БД (используется пока для понравившихся курсов).
- 🌐 **Сетевой слой на Retrofit** — настроен HTTP-клиент с интерсептором (логирование, добавление заголовков). Хотя конечное API пока не подключено, фундамент заложен — достаточно лишь указать базовый URL и эндпоинты, убрав при этом интерсептор.
- 🔄 **Асинхронность на корутинах и Flow** — все длительные операции (запросы к БД, работа с сетью) выполняются в корутинах, для реактивного получения данных используется Flow.
- 🖼 **UI на XML** — интерфейс свёрстан классическим способом (XML), используются кастомные View, RecyclerView с адаптерами.
- 🧩 **Гибкая подмена данных** — из-за отсутствия реальных изображений в моковых данных, все картинки и логотипы заменены на плейсхолдеры. Это показывает умение работать с условиями, когда данные неидеальны.
- 🧪 **Экран-заглушка профиля** — реализован RecyclerView с моковыми данными, но без ViewModel (в доработке).
- 📲 **Навигация через Koin** — управление зависимостями вынесено в DI, что упрощает тестирование и замену реализаций, навигация осуществляется по средствам Jetpack Navigation Component.

## 🚧 Статус проекта и планы по доработке

Проект находится в стадии активной разработки, но уже имеет работающий каркас. В ближайшее время планируется:

- Подключение реального REST API (достаточно будет добавить эндпоинты в Retrofit-сервис).
- Расширение функциональности экрана профиля (вместо заглушки — полноценный экран с данными из сети и БД).
- Добавление экранов для работы с компаниями/карточками (согласно изначальному ТЗ, сейчас это на стадии mock-данных).
- Улучшение обработки ошибок и состояний загрузки (использование sealed классов, UI-состояний).
- Написание unit-тестов для Use Cases и ViewModel, инструментальных тестов для DAO.

**На данный момент проект готов для демонстрации подхода к организации кода, работы с архитектурой и современными библиотеками.**

## 🚀 Как запустить проект

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/test-project.git
Откройте проект в Android Studio (рекомендуется версия Hedgehog или новее).

Синхронизируйте Gradle-зависимости.

Запустите приложение на эмуляторе или устройстве (min SDK 21+).

Важно: Для работы сетевого слоя в будущем потребуется указать базовый URL API в файле gradle.properties или в BuildConfig. Пока можно тестировать с моковыми данными.
